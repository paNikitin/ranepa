КОНТЕКСТ (краткая выжимка из исходного todo)
================================================================================

Цель: универсальный шаблон вайбкодинг-мероприятия, где участники
«через чат» (без терминала, без git, без Docker) собирают веб-эппу с
iOS-look-and-feel и публикуют её на отдельном подпути базового домена.
Первая инстанциация — РАНХиГС, домен ranepa.gigaparsers.ru, 5 слотов.

Шаблон event-agnostic: имя домена, брендинг, число участников и слотов —
переменные конфига. Для будущего мероприятия меняем секцию параметров +
переименование ns/host.

Архитектура зеркалит naisty-look:
  - edge nginx (5.188.118.125 docker prom_nginx_1) терминирует TLS,
    проксирует на traefik k3s 10.43.58.167:10443 с proxy_ssl_verify off;
  - в кластере: ranepa (lending), ranepa-tools (code-server'ы),
    ranepa-app1..5 (приложения участников);
  - образы — Harbor harbor.parsers360.ru:10443/ranepa/*;
  - traefik отдаёт self-signed на unknown SNI.

Параметры РАНХиГС-инстанса (зафиксированы):
  EVENT_NAME=ranepa  DOMAIN=ranepa.gigaparsers.ru  SLOTS=5
  SLOT_PREFIX=app    NAMESPACE=ranepa             SLOT_NS=ranepa-app
  TOOLS_NS=ranepa-tools   HARBOR_PROJ=ranepa      BRAND=РАНХиГС

Решения по разветкам (q3-q5) приняты дефолтами:
  q3 = React 19 + Vite 6 + TS strict + Tailwind 4 + минимальный stack-router
  q4 = без бэкенда (LocalStorage; FastAPI/LiteLLM описаны в AGENTS.md
       как опция «попроси агента включить»)
  q5 = лендинг с N карточками на корне домена

    infra/code-server/sing-box.template.json с плейсхолдерами.

ФАЗА 6. Среда участника (code-server pod'ы)   ✓ DONE 2026-05-10
================================================================================

УПРОЩЕНИЕ vs первоначальный план: вместо code-server (VS Code в браузере) —
ttyd с одним TUI-чатом. Участник видит ровно один интерфейс — диалог с
Claude. Никакого VS Code, никакого shell. Доступ через подпуть, не
поддомен: https://ranepa.gigaparsers.ru/dev/app<N>. Это убирает 5
отдельных DNS-записей и сужает серт до одного домена.

Образ infra/code-server/ — node 20-bookworm + docker-cli + kubectl + gh +
git + ttyd 1.7.7 + sing-box 1.10.7 + Claude Code CLI. start.sh:

  - копирует Claude OAuth кред из секрета в writable ~/.claude/
    (без этого Claude думал что first-run и просил login);
  - pre-seed'ит ~/.claude.json с hasCompletedOnboarding=true,
    bypassPermissionsModeAccepted=true,
    projects."/home/coder/work".hasTrustDialogAccepted=true,
    hasCompletedProjectOnboarding=true (без них участник видел
    welcome wizard и safety prompt'ы);
  - монтирует .dockerconfigjson из harbor-creds в ~/.docker/config.json
    (auto-login к Harbor, без интерактивного docker login);
  - запускает sing-box bootstrap → kill → fresh restart (первый
    sing-box после старта pod'а почему-то впадал в degraded state);
  - sing-box с nohup + redirect + disown (иначе exec ttyd ломал stdio
    sing-box'а после replacement bash → ttyd);
  - ждёт DinD-сокет, делает `git fetch origin && reset --hard origin/main`,
    запускает claude в TUI с --dangerously-skip-permissions.

Манифесты infra/k8s/code-server/{deployment,service,ingress}.yaml:
  - PVC 5Gi на /home/coder/work
  - **DinD sidecar** (docker:27-dind, privileged, runAsUser=0,
    --insecure-registry=harbor.parsers360.ru:10443) с emptyDir socket'ом;
    postStart hook chmod'ит сокет 666 для coder uid=1000;
  - mount: kubeconfig-app<N>, claude-credentials, harbor-creds-ranepa,
    vless-config — все из k8s secrets;
  - ingress на /dev/app<N> с traefik basic-auth + stripPrefix;
  - WebSocket headers в edge nginx (Upgrade/Connection + 3600s timeouts).
