# Шаблон вайбкодинг-мероприятия

Это стартер для веб-приложения с iOS-look-and-feel. Вы (участник
мероприятия) общаетесь только с ИИ-агентом в чате — он сам всё пишет,
собирает и публикует. Тут лежит то, что агент использует как опору.

Первая инстанциация — РАНХиГС, домен `ranepa.gigaparsers.ru`.
Каждому участнику выделен свой подпуть `/app1`, `/app2`, … и свой
дев-чат на `app<N>-dev.ranepa.gigaparsers.ru`.

## Если вы участник

1. Откройте ссылку, которую вам прислали:
   `https://app<N>-dev.ranepa.gigaparsers.ru` (где `<N>` — ваш номер).
2. В открывшемся чате напишите, какое приложение вы хотите. Например:
   - «Сделай чек-лист для подготовки к семинару».
   - «Сделай словарь экономических терминов с поиском».
   - «Сделай тест на знание Конституции на 10 вопросов».
3. Через минуту-две агент напишет ссылку на ваше приложение —
   `https://ranepa.gigaparsers.ru/app<N>/`. Откройте на iPhone в
   Safari, нажмите «Поделиться → На экран Домой» — иконка появится
   как у обычного приложения.
4. Дальше можно править: «сделай заголовок крупнее», «добавь кнопку
   Поделиться», «поменяй синий цвет на зелёный».

Готовые формулировки запросов — в [`examples/prompts.md`](examples/prompts.md).

Если что-то совсем сломалось — попросите агента: «всё сломалось,
сбрось до шаблона».

## Если вы организатор / разработчик

Документация по архитектуре, домену и k8s — в `todo.txt` в корне
репозитория. Спецификация работы агента — в [`AGENTS.md`](AGENTS.md).

Локальный запуск шаблона:

```sh
cd app
npm install
npm run dev
```

Откроется на `http://localhost:5173/`. Vite напечатает локальный
IP — его можно открыть с iPhone в той же Wi-Fi сети.

Сборка прод-образа (для конкретного слота):

```sh
APP_SLUG=app1 ./scripts/deploy.sh
```

(Требует docker, kubectl с настроенным kubeconfig для слота, доступ
в Harbor.)

## Стек

- React 19 + Vite 6 + TypeScript строгий
- Tailwind 4 (CSS-first, через `@tailwindcss/vite`)
- LocalStorage для данных по умолчанию (бэкенда нет)
- Сборка → docker → Harbor → k3s + traefik → edge nginx
