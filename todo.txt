# План: «ranepa» — шаблон для вайбкодинг-мероприятия РАНХиГС
#
# Цель: дать 5 профессорам РАНХиГС инструмент, чтобы они «через чат»
# (без терминала, без git, без Docker) собрали себе веб-эппу с
# iOS-look-and-feel и опубликовали её на отдельном поддомене / подпути
# домена ranepa.gigaparsers.ru.
#
# Ключевая особенность (зафиксировано после уточнений):
#   ▸ профессор взаимодействует ТОЛЬКО с чатом ИИ-агента;
#   ▸ агент сам клонирует шаблон, правит код, билдит, пушит в Harbor,
#     делает kubectl apply, отвечает «готово, см. <url>»;
#   ▸ репозиторий — публичный GitHub юзера (создаётся отдельно),
#     там лежит шаблон + инструкции для агента (AGENTS.md/CLAUDE.md);
#   ▸ один шаблон → 5 форков/веток/копий, по одному на профессора.
#
# Архитектурно зеркалим naisty-look:
#   - edge nginx (5.188.118.125) терминирует TLS, проксирует на traefik k3s;
#   - в кластере namespace ranepa (или 5 разных — см. q1), deploy+svc+ingress;
#   - образы в Harbor harbor.parsers360.ru:10443/ranepa/...;
#   - traefik отдаёт self-signed на unknown SNI, edge nginx проксирует
#     с proxy_ssl_verify off (паттерн naisty-look).

[?] q1. Изоляция приложений 5 профессоров
        А) 5 неймспейсов: ranepa-prof1 … ranepa-prof5. Свой ingress
           на свой host: prof1.ranepa.gigaparsers.ru. Минус: wildcard-cert
           или 5 отдельных certbot-выпусков, 5 server-блоков в edge nginx.
        Б) 1 неймспейс ranepa, 5 deployment+svc, ingress по подпути:
           ranepa.gigaparsers.ru/prof1 … /prof5. 1 серт, 1 server-блок.
           Минус: vite/spa-роутинг придётся учить про base path
           (vite.config: base: '/prof1/'). И профессор должен помнить,
           что его «корень» это /prof1/.
        В) 1 неймспейс, 1 deployment, профессор делает PR в общий
           репозиторий, и контент собирается как мультистраничный сайт.
           Самое простое инфраструктурно, но плохо подходит под
           «5 разных приложений», получится «5 разделов одного сайта».
        Дефолт — Б (подпути). Серт один, edge-nginx один блок,
        DNS — одна запись. Объяснить агенту в AGENTS.md как
        выставить vite base из переменной окружения PROF_SLUG.

[?] q2. Где живёт агент-кодер
        А) Claude Code на ноуте профессора. Дать .zip с готовым
           проектом + .claude/settings.json + Harbor/kubeconfig.
           Профессор запускает `claude` в папке, пишет в TUI.
           Минус: всё-таки нужно установить Claude Code и Node — это
           не «только чат» в чистом виде, но ближайшее, что есть.
        Б) Claude Code в дев-контейнере, доступ через
           code-server / GitHub Codespaces / Gitpod в браузере.
           Профессор открывает ссылку, видит чат, пишет.
           Дороже по хостингу, но НЕ требует установки чего-либо.
        В) Кастомный web-UI на ranepa.gigaparsers.ru/admin,
           внутри Claude Agent SDK, агенту через MCP подключены
           git/docker/kubectl. Самый «product-grade» вариант, но
           требует написать сам web-UI и настроить песочницу.
        Г) Lovable/v0/bolt — отдать на сторонний сервис, потом
           перетянуть собранный код к себе. Самый низкий порог
           для профессора, но теряем контроль над стеком и инфрой.
        Дефолт — Б (Claude Code в дев-контейнере с code-server).
        Поднимем 5 контейнеров (по одному на профессора) с
        предустановленным Claude Code, токенами Harbor + kubeconfig
        с RBAC только на свой ns + ssh-deploy-key к репе.
        Подтверждение НУЖНО — это меняет объём работы значительно.

[?] q3. Стек шаблона
        А) React + Vite + TS + Tailwind (q2-дефолт из v1 плана).
        Б) Svelte 5 + TS — как naisty-look, переиспользуем 1-в-1.
        В) Vanilla HTML + Alpine.js — нулевой порог входа для агента,
           но ограничивает сложность. Для «5 простых демо-эппов» —
           может быть достаточно.
        Дефолт — А (React + Vite + TS + Tailwind). Claude в 2026 пишет
        этот стек идиоматично без подсказок, и он самый
        AI-агент-friendly из перечисленных.

[?] q4. Бэкенд
        А) Без бэкенда. Только LocalStorage + статика. Достаточно
           для 90% «учебных» эппов (опросник, чек-лист, словарь).
        Б) Минимальный FastAPI с /api/echo, без БД. Профессор может
           попросить агента «добавь ручку для X» — агент допишет.
        В) FastAPI + LiteLLM — даём ИИ-чат-ручку «из коробки» (через
           litellm.openclaw.svc, как у naisty-look). Профессор может
           за 10 минут сделать «помощник по моему курсу».
        Дефолт — А для шаблона, опция В описана в AGENTS.md
        («попроси агента включить ИИ-чат — он добавит /api/ai/chat»).
        Бэкенд лишний для большинства учебных эпп, и его отсутствие
        упрощает деплой (просто статика-в-nginx, nodebackend, no PVC).

[?] q5. Что выкладываем на корень ranepa.gigaparsers.ru/
        А) Лендинг мероприятия: «5 приложений профессоров», 5 кнопок
           ведут на /prof1 … /prof5. Минимум 1 час работы.
        Б) Просто redirect на /info или показ 404 — корень не нужен.
        Дефолт — А, лендинг. Дополнительно: показывать билд-статус
        каждого приложения (зелёный/красный кружок).
