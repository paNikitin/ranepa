# План: универсальный шаблон вайбкодинг-мероприятия
#
# Цель: дать участникам мероприятия (преподаватели, студенты, сотрудники
# компании, клуб энтузиастов — кто угодно) инструмент, чтобы они «через
# чат» (без терминала, без git, без Docker) собрали себе веб-эппу с
# iOS-look-and-feel и опубликовали её на отдельном подпути базового
# домена мероприятия.
#
# Шаблон event-agnostic: имя домена, брендинг, число участников,
# названия слотов — переменные конфига. Первая инстанциация — для
# РАНХиГС: домен ranepa.gigaparsers.ru, 5 участников, бренд РАНХиГС.
#
# Ключевая особенность (зафиксировано после уточнений):
#   ▸ участник взаимодействует ТОЛЬКО с чатом ИИ-агента;
#   ▸ агент сам клонирует шаблон, правит код, билдит, пушит в Harbor,
#     делает kubectl apply, отвечает «готово, см. <url>»;
#   ▸ репозиторий — публичный GitHub юзера,
#     там лежит шаблон + инструкции для агента (AGENTS.md/CLAUDE.md);
#   ▸ один шаблон → N веток (по одной на участника).
#
# Архитектурно зеркалим naisty-look:
#   - edge nginx (5.188.118.125) терминирует TLS, проксирует на traefik k3s;
#   - в кластере namespace ranepa + ranepa-app1..N для приложений;
#   - образы в Harbor harbor.parsers360.ru:10443/ranepa/...;
#   - traefik отдаёт self-signed на unknown SNI, edge nginx проксирует
#     с proxy_ssl_verify off (паттерн naisty-look).

================================================================================
ТЕРМИНОЛОГИЯ
================================================================================

«участник»     — человек, который вайбкодит свою эппу через чат с агентом
                 (synonym: автор приложения; в РАНХиГС-инстансе — профессор)
«агент»        — Claude Code, живущий в дев-контейнере на стороне сервера
«слот»         — заранее выделенный namespace + подпуть + dev-pod под одного
                 участника. Идентификатор слота — APP_SLUG (app1, app2, ...)
«мероприятие» — конкретный деплой (для РАНХиГС: домен ranepa.gigaparsers.ru,
                 5 слотов, бренд РАНХиГС)

        («попроси агента включить ИИ-чат — он добавит /api/ai/chat»).
        Бэкенд лишний для большинства эпп, и его отсутствие
